PART 1: BEGINNER LEVEL - SIMPLE ARRAYS
================================================================================

SECTION 1.1: WHAT IS ARRAY QUERYING?
================================================================================

DEFINITION:
  Searching for documents based on values in array fields

PURPOSE:
  ✓ Find documents containing specific array element
  ✓ Match multiple array values
  ✓ Filter by array size
  ✓ Complex array element matching
  ✓ Nested array searching

ARRAY FIELDS IN STUDENT DATA:
  skills - Array of 3-4 technical skills (Python, SQL, TensorFlow, etc.)
  courses - Array of enrolled courses (CS101, MATH201, DB301, etc.)

BASIC OPERATORS:
  Simple match - { array: value }
  $in - Match any value in list
  $all - Match all values in list
  $size - Match by array length
  $elemMatch - Complex conditions
  $regex - Pattern on array elements


================================================================================
SECTION 1.2: BEGINNER - SIMPLE ARRAY MATCHING
================================================================================

DEFINITION:
  Find documents where array contains specific value
  Works even if value is not first element

SYNTAX:
  { arrayField: value }

HOW IT WORKS:
  MongoDB searches entire array for the value
  Matches if value appears ANYWHERE in array


BEGINNER EXAMPLE #1: Find students with Python skill
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: 'Python' })

Translation:
  Find all students who have 'Python' in their skills array

Result:
  ~30-40 students (Python is popular skill)

Why Works:
  { skills: ['Python', 'SQL', 'TensorFlow'] } ✓ MATCHES
  { skills: ['SQL', 'Python', 'R'] } ✓ MATCHES
  { skills: ['Java', 'R'] } ✗ NO MATCH

Use Case:
  Find who knows specific technology
  Project staffing (need Python developers)


BEGINNER EXAMPLE #2: Find students enrolled in CS101
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ courses: 'CS101' })

Translation:
  Students taking this course

Result:
  ~5-10 students

Use Case:
  Course roster
  Find classmates
  Email class


BEGINNER EXAMPLE #3: Find students with SQL skill
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: 'SQL' })

Result:
  ~20-30 students (database is common)

Use Case:
  Database projects
  Hiring pool


SIMPLE ARRAY MATCHING BENEFITS:
  ✓ Simplest syntax
  ✓ Fast with index
  ✓ Intuitive
  ✓ Works regardless of order
  ✓ Works regardless of position

SECTION 1.3: BEGINNER - $IN AND $NIN WITH ARRAYS
================================================================================

DEFINITION:
  $in: Match if array contains ANY of listed values
  $nin: Match if array contains NONE of listed values

SYNTAX:
  { arrayField: { $in: [value1, value2, ...] } }
  { arrayField: { $nin: [value1, value2, ...] } }

KEY DIFFERENCE:
  Simple match: { array: 'Python' } - ONE specific value
  $in: { array: { $in: ['Python', 'Java'] } } - ANY of multiple


BEGINNER EXAMPLE #1: Find students with Python OR Java
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $in: ['Python', 'Java'] } })

Translation:
  Students with EITHER Python OR Java (or both)

Result:
  Students with either skill

Who Matches:
  { skills: ['Python', 'SQL'] } ✓ Has Python
  { skills: ['Java', 'R'] } ✓ Has Java
  { skills: ['Python', 'Java', 'Go'] } ✓ Has both
  { skills: ['C++', 'Rust'] } ✗ Has neither

Use Case:
  Flexible hiring (any language OK)
  Find students with any data skill


BEGINNER EXAMPLE #2: Exclude specific courses
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ courses: { $nin: ['MATH201', 'MATH301'] } })

Translation:
  Students NOT taking either math course

Result:
  Students avoiding math

Use Case:
  Find non-STEM students
  Exclude specific courses


BEGINNER EXAMPLE #3: Data science skill search
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $in: ['Python', 'SQL', 'R', 'TensorFlow'] } })

Translation:
  Students with ANY data skill

Result:
  Data-capable students

Use Case:
  Find for analytics team

SECTION 2.1: INTERMEDIATE - $SIZE OPERATOR
================================================================================

DEFINITION:
  Match arrays by exact element count

SYNTAX:
  { arrayField: { $size: number } }

IMPORTANT LIMITATION:
  $size matches EXACT count only
  For ranges, must use $expr


INTERMEDIATE EXAMPLE #1: Find students with exactly 3 skills
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $size: 3 } })

Translation:
  Students whose skills array has exactly 3 elements

Result:
  Students with 3 skills (not 2, not 4)

Use Case:
  Find well-rounded students
  Training level assessment


INTERMEDIATE EXAMPLE #2: Find students taking 4 courses
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ courses: { $size: 4 } })

Translation:
  Full course load (exactly 4 courses)

Use Case:
  Full-time status
  Schedule planning


INTERMEDIATE EXAMPLE #3: Find students with many skills (WRONG)
────────────────────────────────────────────────────────────────────────────

❌ WRONG - This doesn't work:
  db.students.find({ skills: { $size: { $gte: 5 } } })
  ERROR: $size doesn't support comparison operators

✅ RIGHT - Use $expr:
  db.students.find({ $expr: { $gte: [{ $size: '$skills' }, 5] } })

Translation:
  Students with 5 or more skills

Result:
  Highly skilled students

Use Case:
  Senior roles
  Complex projects


$SIZE LIMITATION NOTE:
  Only matches exact count
  For ranges use $expr with $size
  For any/all use $in/$all


================================================================================
SECTION 2.2: INTERMEDIATE - $ALL OPERATOR
================================================================================

DEFINITION:
  Match if array contains ALL specified values
  Order doesn't matter

SYNTAX:
  { arrayField: { $all: [value1, value2, ...] } }

KEY DIFFERENCE FROM $IN:
  $in: Any of these values (OR logic)
  $all: All of these values (AND logic)

INTERMEDIATE EXAMPLE #1: Find students with Python AND SQL
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $all: ['Python', 'SQL'] } })

Translation:
  Students who know BOTH Python AND SQL

Result:
  ~15-20 students (data engineers/scientists)

Who Matches:
  { skills: ['Python', 'SQL', 'R'] } ✓ Has both
  { skills: ['SQL', 'Python', 'Go'] } ✓ Has both
  { skills: ['Python', 'Java'] } ✗ Missing SQL
  { skills: ['SQL', 'C++'] } ✗ Missing Python

Use Case:
  Team requirements (must have both)
  Full-stack positions
  Specific role requirements


INTERMEDIATE EXAMPLE #2: Full prerequisites
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ courses: { $all: ['CS101', 'MATH201', 'DB301'] } })

Translation:
  Students taking all three courses

Use Case:
  Advanced track requirements
  Program completion check


INTERMEDIATE EXAMPLE #3: Data science team
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $all: ['Python', 'SQL', 'TensorFlow'] } })

Translation:
  Data science experts (all three skills)

Result:
  Elite team members

Use Case:
  Specialized projects


$ALL vs $IN DECISION:
  Use $all: Both Python AND SQL required
  Use $in: Any programming language OK


================================================================================
SECTION 2.3: INTERMEDIATE - $ELEMATCH OPERATOR
================================================================================

DEFINITION:
  Match array elements matching specific conditions
  Allows complex filtering on array elements

SYNTAX:
  { arrayField: { $elemMatch: { condition } } }

USE CASE:
  When simple matching not enough
  Multiple conditions on same array element


INTERMEDIATE EXAMPLE #1: Skills starting with P
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({ skills: { $elemMatch: { $regex: '^P' } } })

Translation:
  Students with skill starting with P

Result:
  Python, PHP, PowerShell, etc.

Use Case:
  Pattern matching on array elements


INTERMEDIATE EXAMPLE #2: Grade range
────────────────────────────────────────────────────────────────────────────

Query (if grades array exists):
  db.students.find({
    grades: { $elemMatch: { $gt: 90, $lt: 95 } }
  })

Translation:
  Students with at least one grade between 90-95

Use Case:
  Grade range filtering
  Performance tracking

SECTION 3.1: ADVANCED - COMBINED ARRAY QUERIES
================================================================================

ADVANCED EXAMPLE #1: Skills with count and content
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({
    $expr: {
      $and: [
        { $gte: [{ $size: '$skills' }, 3] },
        { $in: ['Python', '$skills'] }
      ]
    }
  })

Translation:
  Students with 3+ skills AND has Python

Result:
  Well-rounded Python developers

Use Case:
  Senior developer roles


ADVANCED EXAMPLE #2: Multiple conditions
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({
    $and: [
      { skills: { $all: ['Python', 'SQL'] } },
      { $expr: { $gte: [{ $size: '$courses' }, 3] } }
    ]
  })

Translation:
  (Python AND SQL) AND (3+ courses)

Result:
  Busy, skilled students

Use Case:
  Complex hiring criteria


ADVANCED EXAMPLE #3: Conditional skill filtering
────────────────────────────────────────────────────────────────────────────

Query:
  db.students.find({
    $expr: {
      $or: [
        { skills: { $all: ['Python', 'SQL', 'TensorFlow'] } },
        { $and: [
            { $gte: [{ $size: '$skills' }, 5] },
            { skills: { $all: ['Python', 'R'] } }
          ]
        }
      ]
    }
  })

Translation:
  Data science experts (Path A) OR
  Highly skilled R/Python users (Path B)

Result:
  Candidates for analytics team

Use Case:
  Multiple career paths

SECTION 3.3: BEST PRACTICES & PERFORMANCE
================================================================================

BEST PRACTICE #1: Use simplest matching
  ✓ { skills: 'Python' }
  ✗ { skills: { $in: ['Python'] } }
  Why: Simpler, clearer

BEST PRACTICE #2: Index array fields
  Command: db.students.createIndex({ skills: 1 })
  Benefit: Fast array searches

BEST PRACTICE #3: Use right operator
  $in: "any of these values"
  $all: "all of these values"
  Check: Which logic you need

BEST PRACTICE #4: Size ranges
  ✓ { $expr: { $gte: [{ $size: '$skills' }, 5] } }
  ✗ { skills: { $size: { $gte: 5 } } }
  Why: $size matches exact only

BEST PRACTICE #5: Combine with indexed fields
  { skills: 'Python', age: { $gte: 20 } }
  Why: Filter by index first


PERFORMANCE TIPS:
  Simple match: Very fast with index
  $in: Fast with index
  $all: Medium, can use index
  $size: Medium, no index
  $expr: Slower, complex evaluation


COMMON MISTAKES:

❌ MISTAKE #1: $size with ranges
  Wrong: { skills: { $size: { $gte: 5 } } }
  Right: { $expr: { $gte: [{ $size: '$skills' }, 5] } }

❌ MISTAKE #2: Confusing $in and $all
  $in: ANY match (or)
  $all: ALL match (and)

❌ MISTAKE #3: Not indexing arrays
  Impact: Slow on large collections
  Fix: db.students.createIndex({ skills: 1 })

❌ MISTAKE #4: Over-complicated queries
  Problem: Hard to maintain, slow
  Fix: Keep simple when possible


================================================================================
SECTION 3.4: DECISION TREE
================================================================================

Need to find array element?
  → Use simple match: { array: value }

Need ANY of multiple values?
  → Use $in: { array: { $in: [v1, v2] } }

Need ALL of multiple values?
  → Use $all: { array: { $all: [v1, v2] } }

Need to exclude values?
  → Use $nin: { array: { $nin: [v1, v2] } }

Need to match by count?
  → Use $size or $expr: { $size: ... }

Need complex conditions?
  → Use $elemMatch or $expr

Need multiple criteria?
  → Combine with $and/$or
